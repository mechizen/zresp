# 開発ルール

## 開発ワークフロー

### 0. 初期化
- 新しいセッションを開始したら、ポート 8000 が空いていることを確認する。
  - macOS / Linux: `lsof -i :8000`
  - Windows: `netstat -ano | findstr :8000`
- プロセスが存在する場合は終了する。
  - macOS / Linux: `kill -9 <PID>`
  - Windows: `taskkill /F /PID <PID>`
- 依存をインストールする(初回 / 依存変更時): `uv sync`

### 1. 機能実装 → 動作確認サイクル
- ユーザーから受けた機能依頼を実装する
- サーバを起動: `uv run uvicorn app.main:app --reload --port 8000`
- `--reload` によりコードを変更するとサーバが自動再起動する
- まず自分で `curl` / ブラウザで動作確認し、問題なければユーザーに確認してもらう
- サイクル: **ユーザーの指示 → 実装 → 自動リロード → 動作確認**

### 2. サブエージェント戦略
- メインコンテキストを汚さないため、リサーチ・探索・並列分析はサブエージェントに委ねる
- 1 サブエージェント = 1 タスク に徹する

### 3. 完了前の検証
- 動作確認なしにタスク完了とマークしない
- 「スタッフエンジニアが承認するか？」と自問する

### 4. エレガントさを要求（バランス重視）
- 非自明な変更では「より優雅な方法はないか？」と一時停止する
- 明白な小さな修正には過剰設計しない

### 5. 自律的バグ修正
- バグレポートを受けたら即修正。ハンドホールドを求めない

## 環境変数の管理
- アプリ設定は環境変数 `ZRESP_` プレフィックス、または `.env`(エージェントが管理)で行う。
- Cloudflare Tunnel のトークン等のシークレットは `.env` に置き、`.env.example` に雛形を残す。
- ユーザーの同意なしに `.env` 内の既存値を勝手に削除・変更しない。

## コードスタイル規約

スタックは **Python + FastAPI**。詳細は `CLAUDE.md` の「コードスタイル規約」を参照。

- 型ヒント必須、`Any` を避ける、外部入力は Pydantic で検証
- ruff(line-length 100、ダブルクォート、インポート整列)に従う
- 命名: 関数/変数 `snake_case`、クラス `PascalCase`、定数 `SCREAMING_SNAKE_CASE`
- Cloudflare のヘッダ名・挙動は https://developers.cloudflare.com/ に準拠(推測しない)

### 完了前に必ず通すコマンド
```bash
uv run ruff check .
uv run ruff format --check .
uv run pytest
```
どちらかでもエラーが出たら完了としない。

## コアプリンシプル

- **手抜きなし**: 根本原因を探す。一時的な修正をしない。
- **最小インパクト**: 必要な箇所だけ変更する。バグを混入させない。
